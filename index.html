PayFlow Database Optimization II

This is my strongest portfolio project for showing how I think like a Data Engineer. I used PayFlow to practice modular ETL design, warehouse-focused data shaping, rerun-safe loading, and clearer separation between transformation logic and analytics outputs.

The goal was to move beyond one-script data cleaning and build something closer to a production-minded workflow: explicit stages, clearer responsibilities, and a structure that remains readable when logic changes.
